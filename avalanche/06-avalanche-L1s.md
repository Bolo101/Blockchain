# Avalanche L1s

An Avalanche L1 is an independent blockchain with its own rules for validator membership, tokenomics, and the execution layer. It has its own validator set, and that set reaches consensus to validate the blockchain.

These specialized L1 blockchains can be tailored to fit specific use cases such as DeFi, NFTs, gaming, and enterprise applications.

An Avalanche L1 can define its own:

- Execution logic.
- Fee structure.
- State.
- Networking.
- Security model.

This independence means that L1s do not need to share execution resources, storage, or networking with other L1s or with the Primary Network.

## Avalanche Network

The Avalanche network is a network of independent but interoperable L1s. These chains can communicate with one another, and assets can be transferred between them while preserving interoperability.

As a result, the Avalanche network can:

- Scale more easily.
- Achieve higher overall throughput.
- Offer lower transaction costs.

By using L1s, Avalanche provides a flexible and scalable ecosystem that can adapt to many different requirements and workloads.

## Technical Perspective

From a technical point of view, Avalanche L1s are useful because they let a project control its own environment instead of competing for resources on a shared chain. A team can choose its own validator set, customize its execution layer, and define how fees and governance should work.

This design gives each L1 more autonomy while still allowing it to remain part of a larger interoperable network. In practice, that means an L1 can optimize for a specific workload without sacrificing the ability to interact with the rest of the Avalanche ecosystem.
